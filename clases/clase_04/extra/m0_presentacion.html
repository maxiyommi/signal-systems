M0 · ¿por qué importa?

El plano viene antes que la obra.

Nadie construye un edificio sin un plano. Con software pasa lo mismo: arrancar a codear sin plan termina en un proyecto que colapsa en M2.

M0 no es un milestone de código. Es el acuerdo del grupo:

  • ¿Qué estamos construyendo?
  • ¿Cómo se organizan las partes?
  • ¿Quién hace qué?
  • ¿En qué orden?

Señales de un plano flojo

  • Nadie sabe qué carpeta toca
  • Tres personas tocan el mismo archivo
  • Issues vagos: "hacer M1"
  • Para M2, el código es un caos

5% de la nota. 100% de la base del resto del TP.

- Analogía arquitecto/obra: funciona bien con ing. de sonido. - Timing: 2 min.
M0 · ¿qué están construyendo?

Una API REST.

Un servidor que recibe pedidos HTTP y devuelve datos en JSON.

Ábranla ahora en el navegador:

https://rir-api.onrender.com/docs

O probenla desde la terminal:

# request
curl https://rir-api.onrender.com/health

# response (JSON)
{
  "status": "healthy",
  "version": "0.1.0",
  "timestamp": "2026-04-21T15:30:00"
}

Endpoint

Una URL que hace una cosa. Ej: GET /health devuelve "estoy vivo".

Request · Response

El cliente manda audio y parámetros; la API responde con T60, C80, etc.

JSON

Formato para pasar datos. {"t60": 1.2, "c80": 5.4}

- ESTE ES EL MOMENTO WOW. Abrir el link en pantalla en vivo. - Si Render está dormido, esperar ~30s — advertirlo. - Ejecutar el curl en vivo también, que vean la respuesta cruda. - Timing: 5 min.
M0 · arquitectura

Tres capas. Memorícenlo.

Routers · puerta de entrada

@router.post("/pink-noise")
async def pink_noise(req: PinkNoiseReq):
    audio = services.generar_ruido_rosa(
        req.duracion, req.fs
    )
    return {"audio": audio.tolist()}

Reciben el request, llaman al service, devuelven JSON. No calculan.

Services · cerebro

def generar_ruido_rosa(
    duracion: float, fs: int
) -> np.ndarray:
    n = int(duracion * fs)
    # algoritmo Voss-McCartney
    return signal

Funciones puras. No saben de HTTP ni de JSON.

Schemas · aduana

class PinkNoiseReq(BaseModel):
    duracion: float = Field(gt=0, le=60)
    fs: int = Field(default=44100)

Validan que los datos entren y salgan con la forma correcta.

- Esta es LA idea central del TP. - "El router no calcula, el service no sabe de HTTP, el schema no hace lógica". - Recorrer los tres snippets: el router LLAMA al service (no lo implementa); el service es una función pura; el schema es declarativo. - Timing: 4 min.

README.md

Lo primero que mira el docente cuando abre el repo.

Debe tener

  • Nombre y descripción (1-2 párrafos)
  • Integrantes con rol asignado
    señales · procesamiento · testing/CI · docs
  • Instrucciones que funcionen copy-paste
  • Estructura del proyecto comentada
  • Branching strategy

Bueno

git clone https://...
cd rir-api
uv sync
uv run uvicorn app.main:app --reload

Flojo

"Para instalar, pip install (ver requirements.txt)"

- Los roles no son decorativos. - "Instrucciones que funcionan" = probálas clonando fresh. - Timing: 3 min.

Entregable 04 / 04

Estructura del repo.

Buena noticia: ya está hecha. Forkean el template, no lo reescriben.

rir-api/
├── pyproject.toml    ← dependencias
├── README.md
├── app/
│   ├── main.py       ← entrada FastAPI
│   ├── settings.py
│   ├── routers/
│   │   └── health.py ← ya funciona
│   ├── schemas/
│   └── services/
├── tests/
│   └── test_placeholder.py
├── docs/
└── data/

Qué hacer

  1. Forkear template_repo
  2. Clonar localmente
  3. Levantar el server:
    uv sync
    uv run uvicorn app.main:app --reload
  4. Abrir localhost:8000/docs
  5. Correr tests: pytest -v

Si todo esto funciona, cumplieron los requisitos técnicos de M0.

- NO están construyendo la app desde cero. El template ya corre. - Si alguien no logra levantar el server: Slack antes del sábado. - Timing: 3 min.
M0 · el único código que ya existe

El endpoint /health.

El más tonto posible: devuelve "estoy vivo". Confirma que el servidor corre.

from datetime import datetime
from fastapi import APIRouter

router = APIRouter()


@router.get("/health")
async def health_check():
    """Verifica que la API esta funcionando correctamente."""
    return {
        "status": "healthy",
        "version": "0.1.0",
        "timestamp": datetime.now().isoformat(),
    }

Criterio de M0: que esto responda 200 OK cuando levantan el server.
